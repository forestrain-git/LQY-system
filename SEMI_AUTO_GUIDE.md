# 半自动化执行指南（可行版）

**原则**：人是架构师和验证者，Claude是高级代码生成器
**目标**：每天4小时有效工作，产出可交付的代码

---

## 每日工作流程

### 09:00-09:30 | 准备阶段（你主导）

1. **查看昨日进度**
   ```bash
   git log --oneline -5
   git status
   ```

2. **准备今日任务清单**
   - 从 `prompts/dayX/` 读取今日Prompt
   - 简化成3-5个具体任务
   - 写在 `TODO_TODAY.md`

3. **启动Claude Code**
   ```bash
   cd /mnt/d/projects/lqy-system
   claude
   ```

### 09:30-11:30 | 编码阶段（Claude主导）

**你输入**：
```
请读取ARCHITECTURE_CONTRACT.md确认规范。

今日任务：
1. [具体任务1]
2. [具体任务2]
3. [具体任务3]

请先生成任务1的代码，完成后告诉我。
```

**Claude执行**：
- 生成代码
- 自测基础功能
- 报告完成

**你做**：
- 偶尔看一眼，不干预
- 如果Claude卡壳超过10分钟，询问情况

### 11:30-12:00 | 验证阶段（你主导）

**必须执行的验证清单**：

```bash
# 1. 代码审查
git diff
# 检查：是否遵循架构契约？代码质量是否可接受？

# 2. 功能测试
cd backend
make test
# 预期：通过率>80%

# 3. 端到端测试（关键！）
curl http://localhost:8000/health
curl http://localhost:8000/api/v1/devices
# 实际调用API，确认功能真能用

# 4. 检查覆盖率
pytest --cov=app --cov-report=term-missing
# 新代码覆盖率>80%
```

**决策点**：
- [ ] 全部通过 → 继续下阶段
- [ ] 部分失败 → 给Claude反馈，要求修复
- [ ] 严重问题 → 回退到上次提交

### 12:00-13:30 | 午休

### 13:30-15:30 | 修复/继续阶段（Claude主导）

根据上午验证结果：

**如果上午通过**：
```
上午验证通过。

继续任务：
4. [任务4]
5. [任务5]
```

**如果上午有问题**：
```
上午验证发现问题：

1. [具体问题1，如：测试test_create_device失败，返回404]
2. [具体问题2]

请修复后重新测试。
```

### 15:30-16:00 | 最终验证（你主导）

重复验证清单，确保：
- 所有今日功能可用
- 测试通过
- Git工作区干净

### 16:00-16:30 | 提交+总结（你主导）

```bash
# 1. 提交
git add .
git commit -m "Day X: [今日完成内容]

- [完成点1]
- [完成点2]
- 测试通过率：XX%

co-authored-by: Claude Code"

# 2. 打标签
git tag -a dayX-complete -m "Day X completed"

# 3. 推送
git push origin main --tags
```

**写明日计划**：
```markdown
# Day X+1 计划

## 今日完成
- [完成的内容]

## 明日目标
- [目标1]
- [目标2]

## 技术债/待修复
- [如果有]
```

### 16:30 | 结束

**重要**：/clear 清理Claude上下文，或关闭终端。
这样明天启动时是干净状态。

## 沟通技巧

### 对Claude说这些效果好：

| 你说 | 效果 |
|------|------|
| "先读取ARCHITECTURE_CONTRACT.md" | 确保遵循规范 |
| "生成完成后，先自测再报告" | 减少明显错误 |
| "这部分有问题：[具体错误]" | 精准反馈，快速修复 |
| "从上次检查点重来" | 避免错误累积 |

### 避免说这些：

| 避免 | 原因 |
|------|------|
| "继续"（没有上下文） | Claude不知道继续什么 |
| "修复所有问题" | 太笼统，Claude可能乱改 |
| "快一点" | 质量下降 |

## 应急预案

### 场景1：Claude生成代码完全跑偏
**症状**：代码不符合架构契约，或根本跑不通
**处理**：
```bash
git checkout dayX-1-checkpoint  # 回退到上次检查点
```
然后告诉Claude："回退到检查点，我们重新做[任务]，注意[具体问题]"

### 场景2：Claude陷入循环（反复修复同一问题）
**症状**：同一个错误修3次还没好
**处理**：
- 人工介入，查看具体问题
- 或简化需求，绕过该问题
- 或标记为技术债，明天处理

### 场景3：下午4点还有功能没完成
**处理**：
- 优先保证已完成功能的测试和提交
- 未完成的功能拆到明天
- 不要赶工，质量>速度
