# 龙泉驿环卫智能体 - 启动器


## 启动协议

### Step 0: 初始化检查

**执行**：
1. 检查当前工作目录
2. 检查Python版本（3.10+）
3. 检查Docker状态
4. 检查端口占用（8000, 5432, 6379）

**输出格式**：
```
启动检查
========
工作目录: [路径]
Python: [版本] [OK/失败]
Docker: [状态] [OK/失败]
端口8000: [空闲/占用] [OK/失败]
端口5432: [空闲/占用] [OK/失败]
端口6379: [空闲/占用] [OK/失败]

结论: [可以开始/需要修复]
修复建议: [如果有问题]
```

**如果检查失败**：
- 输出具体错误
- 给出修复命令
- 等待用户确认"已修复，继续"

## 上下文压缩机制

### 何时压缩

当上下文长度超过150K tokens（约75%容量），执行压缩。

### 压缩方式

**保留内容**：
- ARCHITECTURE_CONTRACT.md（必须始终记住）
- 当前任务的最终代码（而非中间迭代）
- 未解决的技术债清单
- 用户的明确指令

**清理内容**：
- 已完成的步骤的详细执行过程
- 已修复的错误记录
- 成功的测试输出

**压缩后输出**：
```
[上下文压缩]
===========
保留: 架构契约, Day 1 Step 3最终代码, 未解决: 测试X待修复
清理: Step 1-2详细过程, 已修复的错误日志

当前状态: Day 1 Step 3, API实现阶段, 测试通过率80%
继续执行...
```

## 用户快捷指令

任何时候，用户可以说：

| 指令 | 动作 |
|------|------|
| "继续" | 执行下一步 |
| "检查" | 输出当前进度和文件状态 |
| "重做" | 重新执行当前步骤 |
| "回退" | 回到上一步 |
| "跳过" | 跳过当前步骤（标记技术债） |
| "结束" | 保存当前状态，准备退出 |
| "Day X" | 跳到指定天数 |
| "清理" | 执行上下文压缩 |
| "契约" | 重新读取架构契约 |

## 状态持久化

每次完成一个Step，自动创建状态文件：

`progress.json`:
```json
{
  "current_day": 1,
  "current_step": 3,
  "status": "completed",
  "git_commit": "abc123",
  "tech_debt": ["测试X待修复"],
  "timestamp": "2026-04-06T16:30:00Z"
}
```

这样即使Claude Code崩溃/重启，也能恢复状态。
