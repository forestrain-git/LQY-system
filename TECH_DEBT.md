## 🔴 高优先级 (必须修复)

### TD-001: Kimi API Key配置
- **位置**: `backend/app/modules/ai/service.py`
- **问题**: AI助手需要真实的Kimi API Key才能工作
- **影响**: AI对话功能无法使用
- **修复方案**: 
  1. 从Moonshot AI官网申请API Key
  2. 配置到环境变量 `KIMI_API_KEY`
  3. 重启后端服务
- **预计工时**: 30分钟

### TD-002: 前端API连接
- **位置**: `frontend/src/views/*.vue`
- **问题**: 前端页面使用Mock数据而非真实API
- **影响**: 数据不是实时的
- **修复方案**:
  1. 创建API客户端封装
  2. 替换所有Mock数据为API调用
  3. 添加加载状态和错误处理
- **预计工时**: 4小时

## 🟡 中优先级 (建议修复)

### TD-003: equipment_id外键约束
- **位置**: `backend/app/modules/workflow/models.py:346`
- **问题**: equipment_id外键被注释，无法保证数据一致性
- **影响**: 可能关联不存在的设备
- **修复方案**:
  1. 添加设备模型迁移
  2. 恢复外键约束
  3. 更新相关查询
- **预计工时**: 2小时

### TD-004: 权限验证系统
- **位置**: 全局
- **问题**: API端点没有身份验证
- **影响**: 任何人都可以访问敏感数据
- **修复方案**:
  1. 实现JWT认证
  2. 添加登录/注册API
  3. 配置路由守卫
- **预计工时**: 6小时

### TD-005: 数据库连接池优化
- **位置**: `backend/app/database.py`
- **问题**: 默认连接池配置可能不适合生产环境
- **影响**: 高并发时性能问题
- **修复方案**:
  1. 调整连接池大小
  2. 配置连接超时
  3. 添加连接监控
- **预计工时**: 2小时
